Project Description

Implement Flappy Bird game similar to flappybird.io (you can check live game here: flappybird.great-site.net.

Tools Used

Plain javascript mostly + bootstrap and a little bit of jQuery

Design Details

I tried to create a very simple game engine along the way and created the game using that game engine. It was inspired by GoDot (which could be used for learning game development course instead of pure javascript btw)

So the Idea is that we have the game as objects nesting objects. And objects usually extend from multiple layers of base classes. Similar to DOM For example we have a scene named `GameScene` its class that inherits from `Scene` Class and inside that scene we have an object which can have other objects inside it. Like for example a the FlappyBirdCharacter has multiple children inside it like FlappySprite, Collisions etc

Another important thing to point out is that objects inside the game are generated automatically based on childrenJSON defined on parent `Node`. For example a Scene like GameScene has multiple nested objects inside it and they could have their own children. the object initiations are done automatically using game engine from those childrenJSON arrays. those arrays contains json objects defining name and type of the element (Node) that game engine should create along with properties assosiated with that element (Node) and its childrenJSON which would be populated and generated and initialized too.

So most of the logic regarding drawing objects of different type or handling them is pre written inside code folder and we use them by defining array of childrenJSON inside the scene. But we can also write our own customized Node (element) inside `scripts` which can have multiple children inside it by default or have custom codes for handling different logic and more complex logic and then it can be used as normal default objects defined inside core folder. This way we can add custom Nodes without having to edit the core folder of the game engine. Some examples: ` FlappyBirdCharacter.js `

Base Class for game scenes. This is responsible for handling game loops. It has "process loop" and "physical loop" and "draw loop" and it also runs loops of children objects of the scene

CNode.js

Base class for all objects. So basically evey object including Scene either directly or directly inherits from this class.

it handles things like child initialization (turning childrenJSON into children Objects). defines methods for loops and object lifecycles like "init" "ready" "process" , ... . Handles loops events of objects. calculates positions and dimensions of object. "getChild", "getParentNode", ...

basically everything that we want an object to be able to do. if we want to create our own customized object its enough to extend from this class or a class that inherits from this class and it will do the magic.

CollisionShape2D.js

Extends from `Node2D.js`.

Defines collision area (a rect). This is used by `Area2D.js`, `CharacterBody2D.js`, `StaticBody2D.js` to define collision area(s). On init registers its parent to collidable objects list.

_init(){ const parentNode = this.getParentNode() CollisionObject2D.registerCollision(parentNode, this) }

Sprite2D.js

Extends from `Node2D.js`.

Draws an sprite on canvas using given properties. Supported properties:

const defaultProperties={ origin:{ X:0 , Y:0 }, position:{ X:0, Y:0 }, offset:{ X:0, Y:0 }, offset2:{ X:0, Y:0 }, destinationOffset:{ X:0, Y:0 }, repeat: 'no', scale: 1, scaleMode:'stretch', width:'auto', height: 'auto', rotation: 0, positionMode: 'relative' }

StaticBody2D.js

Extends from `Node2D.js`.

Defines collidable object that is static and cannot move (like walls, ground, etc)

AudioStreamPlayer.js

Camera2D.js

Extends from `CNode.js`.

Defines camera2D. properties defines:

const defaultProperties={ position:{ X:240, Y:320 }, origin:{ X: 0.5, Y: 0.5 }, viewWidth: 480, viewHeight: 640, showCameraRect: true, default: false }

the camera itself does not do much but others will use its properties to adjust themselves or scene can change position of camera based on game logic etc
